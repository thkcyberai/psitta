## 3. Flagged for founder decision (NOT changed — outside version-bump scope)

1. **Installer metadata narrative (pubspec `msix_config.description`):** still reads `"Reading to Listening - ultra-natural document narration"`, and the pubspec package `description` reads `"Ultra-natural document narration. Listen to any document."` Both are Reading-era copy; the msix description ships inside the 2.0.0 installer metadata. Changing narrative copy is beyond a version bump — say the word and it becomes a one-line follow-up before the RC build (recommended: align with the Writing Nook platform line).
2. **RC signing (blocking decision for RC validation):** `sign_msix: false` produces an **unsigned** MSIX, and Windows will not install an unsigned package — so installation/uninstall/identity smoke rows cannot run without a signature. The dev certificate (`psitta-dev.pfx`) may be used to sign the **local RC artifact only** — that artifact is quarantined and never published; the public release is rebuilt/signed with the production certificate per the signing tutorial (standing rule M1 intact). Founder approval requested for local-only dev-signing of the RC.
3. **Upgrade path (1.1.2.0 → 2.0.0.0) cannot be truly validated pre-publish:** the installed 1.1.2.0 base is production-signed; a dev-signed RC is not a valid upgrade source for it. The adapted procedure moves upgrade-path validation into a **staged publish** step: after GO + production signing, upload the MSIX to `releases/2.0.0.0/` only, validate the upgrade on one machine by opening the updated appinstaller directly, and only then replace the public root `psitta.appinstaller`. No user receives the release before the upgrade path is proven.
